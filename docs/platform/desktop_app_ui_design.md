## 1. 设计理念

### 1.1 核心原则

**以"快速成功"为第一目标**
- 新用户 3 分钟内必须体验到价值
- 每个页面只有一个主要行动点
- 减少选择，减少思考

**渐进式披露（Progressive Disclosure）**
- 首次使用：只看到"开始录制"
- 有工作流后：看到"最近使用"
- 深度用户：看到"全部管理"

**即时反馈**
- 每个操作都有即时响应
- 进度可视化
- 错误友好提示

### 1.2 用户旅程设计

```
新用户：
  首页 → 开始录制 → 录制引导 → 生成中 → 预览 → 自动执行 → 结果 → 引导保存

老用户：
  首页 → 看到最近使用 → 点击运行 OR 点击新建录制
```

## 2. 页面结构

### 2.1 信息架构

```
首页（Home）
├─ 快速启动区域（主要）
├─ 最近使用区域（次要，有数据时才显示）
└─ 底部导航（快速录制、我的工作流、录制数据、对话生成）

录制流程页面（Recording Flow）
├─ 录制引导（首次）
├─ 录制中
├─ 生成中
├─ 预览确认
└─ 执行监控

Workflow 管理页面（Workflow List）
├─ 分类筛选（草稿、可用、失效）
├─ 工作流卡片列表
└─ 搜索

录制数据页面（Recordings Library）⭐ 新增
├─ 录制列表（缩略信息）
├─ 筛选和搜索
└─ 快捷操作（查看详情、生成工作流、删除）

Workflow 详情页面（Workflow Detail）
├─ 基本信息
├─ 步骤列表（可测试）
├─ 对话调整区
└─ 执行按钮

录制详情页面（Recording Detail）⭐ 新增
├─ 录制元数据
├─ 操作时间线
├─ 提取的字段列表
└─ 生成工作流按钮

对话式生成页面（Conversational Generation）⭐ 新增
├─ AI 对话区域（主要）
├─ 录制引用侧边栏（可选）
└─ 生成的工作流预览

执行监控页面（Execution Monitor）
├─ 浏览器窗口（实时）
├─ 执行日志
└─ 进度条

结果展示页面（Result View）
├─ 数据预览
├─ 下载选项
└─ 后续引导
```

#### 3.2.2 录制中界面

**设计：浮动小窗 + Chrome 全屏**

```
Chrome 浏览器（用户正常操作）






              ┌─────────────────────────┐
              │ 🔴 正在录制...           │
              │                          │
              │ 已记录 8 个操作          │
              │                          │
              │ 💡 提示：选中数据并复制  │
              │                          │
              │ ┌─────────────────────┐ │
              │ │   ⏹ 停止录制         │ │
              │ └─────────────────────┘ │
              │                          │
              │ [− 最小化]               │
              └─────────────────────────┘
              ↑ 右下角浮动窗口（可拖动）
```

**设计要点：**
1. **浮动小窗**:
   - 默认位置：右下角
   - 可拖动，记住用户位置
   - 半透明背景，不遮挡内容
   - 可最小化成小圆点

2. **实时提示**:
   - 显示已记录操作数量
   - 智能提示："还没选中数据？试试选中并复制"
   - 记录到 `copy_action` 后，提示变为"✅ 已记录数据字段"

3. **停止录制按钮**:
   - 醒目，大号按钮
   - 点击后立即进入生成流程

   - 显示倒计时："3 秒后自动运行..."

```
│ ┌────────────────────────────┐│                                │
│ │ 💬 继续对话...              ││                                │
│ │ [发送]                     ││                                │
│ └────────────────────────────┘│                                │
│                                │                                │
└────────────────────────────────┴────────────────────────────────┘

生成完成后，跳转到预览页面（类似快速启动的预览界面）
```

**设计要点：**

1. **布局**:
   - **左侧（主要）**: AI 对话区域，占 70% 宽度
   - **右侧（可选）**: 录制引用侧边栏，占 30% 宽度
   - 侧边栏可关闭，关闭后对话区占满全屏

2. **AI 对话区域**:
   - **聊天界面**: 类似微信/ChatGPT 的对话框
   - **消息气泡**: 用户（右侧蓝色）、AI（左侧灰色）
   - **时间戳**: 每条消息显示时间
   - **示例提示**: 首次使用显示示例任务
   - **输入框**: 底部固定，支持多行输入
   - **实时生成**: AI 回复支持流式输出（打字效果）

3. **录制引用侧边栏**:
   - **搜索框**: 快速查找录制
   - **录制卡片**: 显示录制名称、操作数、字段数
   - **引用按钮**: 点击"+ 引用"后，录制信息会插入对话中
   - **关闭按钮**: 可隐藏侧边栏，节省空间

4. **引用录制后的效果**:
   - 用户点击"+ 引用"后，自动在输入框插入：
     ```
     📎 引用录制：抓取商品信息录制
     请基于这个录制生成工作流
     ```
   - AI 可以读取录制的操作序列和字段信息
   - 生成更准确的工作流

5. **交互流程**:
   - **场景 1 - 纯对话**:
     - 用户描述任务 → AI 提问澄清 → AI 生成工作流 → 预览页面

   - **场景 2 - 引用录制**:
     - 用户选择录制 → 点击"引用" → AI 自动理解录制 → 生成工作流 → 预览页面

   - **场景 3 - 混合方式**:
     - 用户描述任务 → AI 建议："我看到你有类似的录制，要引用吗？" → 用户确认 → AI 生成

6. **生成完成后**:
   - 跳转到工作流预览页面（复用快速启动的预览界面）
   - 显示生成的工作流摘要
   - 提供"立即运行"和"对话调整"选项

**底部导航栏设计：**

```
┌─────────────────────────────────────────────────────┐

│                                                       │

│  (当前选中时加粗显示下划线)                           │
│                                                       │
└─────────────────────────────────────────────────────┘
```

**导航逻辑：**
1. **快速录制**: 点击后进入快速启动流程（录制 + 生成 + 执行）
2. **我的工作流**: 进入工作流管理页面（已保存的工作流）
3. **录制数据**: 进入录制数据页面（历史录制数据，可查看详情和生成工作流）
4. **AI生成**: 进入对话式生成页面（纯聊天生成工作流）

**功能关联设计：**

**三种 Workflow 生成路径可视化：**
```
路径 1: 快速录制路径
  快速录制 → 录制中 → 生成工作流 → 预览 → 执行 → 结果

路径 2: 录制库路径
  录制库 → 选择录制 → 查看详情 → 生成工作流 → 预览 → 执行 → 结果

路径 3: AI 对话路径
  AI生成 → 对话描述任务 → (可选引用录制) → 生成工作流 → 预览 → 执行 → 结果
```

**页面跳转关系：**
```
首页 / 底部导航
├─ [快速录制] → 快速启动流程（录制 → 生成 → 预览）
├─ [我的工作流] → 工作流管理页 → 工作流详情页
├─ [录制数据] → 录制数据页面 → 录制详情页
└─ [AI生成] → 对话式生成页面

录制数据页面
├─ 查看详情 → 录制详情页
└─ 生成工作流 → 预览页面（快速生成，跳过详情）

录制详情页
└─ 生成工作流 → 预览页面

对话式生成页面
├─ 引用录制 → 弹出录制选择器（从录制数据中选择）
└─ 生成完成 → 预览页面

所有生成路径最终汇聚：
  预览页面 → 执行监控页面 → 结果展示页面 → 保存工作流
```

## 4. 交互流程

### 4.1 新用户首次使用完整流程

```
1. 打开 App
   → 看到首页（新用户版本）

2. 点击"开始录制"
   → 弹出录制教程（3 页，可跳过）

3. 看完教程 / 跳过教程
   → 开始录制
   → Chrome 打开，浮动小窗显示"正在录制..."

4. 用户正常操作
   → 浮动小窗实时提示："还没选中数据？试试选中并复制"

5. 用户选中文本并复制
   → 浮动小窗提示更新："✅ 已记录数据字段"

6. 用户继续操作
   → 浮动小窗显示："已记录 8 个操作"

7. 用户点击"停止录制"
   → 进入生成页面（全屏，进度条）
   → 显示："正在理解你的操作流程... 60%"

8. 生成完成（约 20 秒）
   → 进入预览页面
   → 显示工作流摘要
   → 倒计时："3 秒后自动运行..."（用户可取消）

9. 自动开始执行 / 用户点击"立即运行"
   → 进入执行监控页面
   → 上半部分：浏览器窗口（实时）
   → 下半部分：执行日志（滚动）
   → 底部：进度条

10. 执行完成
    → 进入结果展示页面
    → 显示："🎉 执行成功！已抓取 25 条数据"
    → 数据预览表格
    → 引导："💡 下次想用？保存这个工作流"

11. 用户点击"保存工作流"
    → 保存成功提示
    → 引导："✨ 恭喜完成首个自动化！节省了 30 分钟"
    → 提供后续选项："再运行一次" "录制新工作流" "返回首页"
```

**关键时间节点：**
- 0:00 - 打开 App
- 0:30 - 开始录制
- 3:00 - 停止录制
- 3:20 - 生成完成
- 3:23 - 开始执行（自动）
- 5:00 - 执行完成
- 5:30 - 保存工作流

**总计：5 分 30 秒完成首次体验**

### 4.3 调试工作流流程

```
1. 从首页进入"查看全部工作流"
   → Workflow 管理页面

2. 找到草稿状态的工作流
   → 点击"继续调试"
   → 进入 Workflow 详情页

3. 查看步骤列表
   → 发现步骤 2 有警告："⚠️ 部分测试（缺少库存字段）"

4. 点击步骤 2 的"测试"按钮
   → 弹出单步测试界面
   → 选择"使用上次执行的状态"
   → 点击"开始测试"

5. 测试完成
   → 看到结果："缺少库存字段"
   → 在测试界面的对话框输入："增加库存字段"
   → 点击"发送"

6. LLM 调整工作流
   → 显示："已在步骤 2.1 添加库存字段"
   → 点击"重新测试"

7. 再次测试
   → 成功，显示："✅ 执行成功，已包含库存字段"
   → 点击"确认通过"

8. 回到详情页
   → 步骤 2 状态更新为："✅ 测试通过"
   → 点击"执行工作流"

9. 完整执行
   → 进入执行监控页面
   → 执行完成
   → 查看结果
   → 保存
```
